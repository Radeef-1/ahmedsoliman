# دليل تهيئة البنية التحتية والعمليات (DevOps & SaaS Architecture Guide)

يوضح هذا الدليل معمارية عزل الشركات المتعددة (SaaS Multi-Tenancy) المعتمدة للتطبيق، وخطة النشر على الإنترنت (Cloud Deployment)، بالإضافة إلى تقسيم المهام بين ما يمكن للمساعد الذكي إعداده تلقائياً وما يجب عليك فعله يدوياً.

---

## 🏗️ معمارية عزل الكيانات والشركات (SaaS Multi-Tenancy Isolation)

تم تعديل وتطبيق معماري **عزل قاعدة البيانات لكل مستأجر (Database-per-Tenant)** كالتالي:
1. **قاعدة البيانات الدليلية (Global Directory Database):** ملف `global.json` يقوم فقط بتخزين بيانات التسجيل الأساسية للشركات (المعرف الرقمي، البريد الإلكتروني، الاسم، وهاش كلمة المرور) للتحقق عند تسجيل الدخول.
2. **قواعد البيانات المعزولة (Isolated Company Databases):** عند تسجيل أي شركة جديدة، يتم إنشاء ملف قاعدة بيانات منفصل وخاص بها في المجلد `backend/db/` تحت اسم `company_{company_id}.json`.
3. **العزل الفعلي:** أي عمليات قراءة، كتابة، حساب بدلات، رفع إكسل، أو استعلامات مشاريع تتم **فقط** على ملف الشركة المعنية ولا يمكن لعمليات شركة (A) الوصول في الذاكرة أو القرص لملفات شركة (B) نهائياً.

---

## 🛠️ تقسيم المسؤوليات لتشغيل التطبيق على الإنترنت (Who Does What?)

بناءً على طلبك، تم تقسيم المسؤوليات لتسريع عملية الرفع السحابي:

### 1. ما قمت بإعداده برمجياً بدون تدخل منك (Done by Agent):
* [x] **برمجة معمارية العزل الفعلي (Tenant Isolation):** فصل قواعد البيانات في ملفات مستقلة في الكود.
* [x] **إعداد ملف Dockerfile للواجهة الخلفية (backend/Dockerfile):** لبناء الحاوية وتجهيز خادم Node.js للتشغيل في أي بيئة سحابية.
* [x] **إعداد ملف Dockerfile وملف Nginx للواجهة الأمامية (frontend/Dockerfile):** لبناء تطبيق React وخدمته عبر خادم ويب Nginx سريع وآمن ومعالجة مسارات الـ SPA.
* [x] **ملف تركيب الحاويات (docker-compose.yml):** لربط خادم الواجهة الخلفية والواجهة الأمامية بحجم تخزين مشترك (Volume) لحفظ ملفات قواعد البيانات المعزولة محلياً أو سحابياً بأمر واحد: `docker-compose up --build`.
* [x] **إعداد خط أنابيب التطوير المستمر (CI/CD Workflow):** ملف GitHub Actions ليقوم بالتحقق التلقائي وبناء التطبيق عند رفع الكود على GitHub.

### 2. ما يجب عليك القيام به يدوياً (Must be Done by User):

#### أولاً: تسجيل الحسابات السحابية (Accounts Registration)
* قم بإنشاء حساب على إحدى منصات الاستضافة السحابية المناسبة لك، مثل:
  * **خيارات مجانية/رخيصة وسهلة جداً:** [Render.com](https://render.com) أو [Railway.app](https://railway.app).
  * **خيارات متقدمة واحترافية:** [AWS](https://aws.amazon.com) أو [DigitalOcean](https://digitalocean.com).

#### ثانياً: إدخال متغيرات البيئة السرية (Environment Variables) في لوحة الاستضافة
* `PORT`: تعيين المنفذ (مثل 5000 للواجهة الخلفية).
* `JWT_SECRET`: كتابة سلسلة نصية طويلة سرية لتشفير رموز الأمان والتوكينز الخاصة بالشركات (لا تشاركها مع أحد).
* `NODE_ENV`: ضبطها على القيمة `production`.

#### ثالثاً: تهيئة النطاق والعناوين (DNS & Domain Configuration)
* حجز نطاق خاص بالتطبيق (مثال: `mycostapp.com`).
* توجيه الـ DNS الخاص بالدومين ليشير إلى خوادم الاستضافة التي قمت بإنشائها.
* تفعيل بروتوكول الأمان SSL ليكون الرابط `https` لحماية كلمات مرور الموظفين وبيانات الرواتب أثناء النقل.
