# 📋 الخطة التنفيذية المعتمدة والنهائية: الهيكل التنظيمي المطور (SaaS)

تم تطوير هذه الخطة بالكامل لتلبي متطلبات الهيكل التنظيمي للمجموعات والشركات السعودية (الكيانات الكبرى بالفروع التابعة والمشاريع وتوزيع الموظفين)، مع دمج تحسينات الجودة لفلترة البيانات، وتأمينها، ومطابقة ملف استيراد الإكسيل الخاص بالمنشأة [تقرير تكلفة الموظف 2023.xlsx](file:///c:/Users/z2k1s/Music/12522/تقرير%20تكلفة%20الموظف%202023.xlsx) بدقة.

## 1. الهيكل التنظيمي للمجموعة (Organizational Structure)

1. **الكيانات الكبرى (Legal Entities):**
   * الكيانات القانونية الرئيسية للمجموعة (مثال: شركة ريال البركة للتجارة).
   * الحقول: المعرف (ID)، الاسم (Name)، **الرقم الموحد 700** (Unified Number).

2. **الفروع (Branches):**
   * الفروع التشغيلية التابعة للكيانات الكبرى (مثال: فرع الرياض، فرع جدة).
   * الحقول: المعرف (ID)، الاسم (Name)، **رقم السجل التجاري** (CR Number)، **الكيان الكبير التابع له** (Entity ID).

3. **المشاريع (Projects):**
   * يتبع كل مشروع لـ "فرع" معين أو "كيان كبير" معين.
   * الحقول: المعرف (ID)، الاسم (Name)، معرف الكيان التابع له (Entity ID - اختياري)، معرف الفرع التابع له (Branch ID - اختياري).

4. **الموظفون (Employees):**
   * **الفرع القانوني (Legal Branch):** الفرع المسجل عليه الموظف قانونياً (لحساب التأمينات والتوطين للكيان الكبير التابع له).
   * **الفرع المالي (Cost Branch):** الفرع الذي يتحمل تكلفة وراتب الموظف فعلياً (في تقارير المشاريع والفروع المجمعة).
   * **المشروع المنسب له (Project):** المشروع المرتبط بالفرع المالي/القانوني للموظف.
   * **نوع الموظف السعودي:** (سعودي عامل / سعودة لدعم النشاط فقط).

## 2. استراتيجية استيراد ملف الإكسيل المعتمد (Excel Import Strategy)

لمطابقة كشف الإكسيل المرفق [تقرير تكلفة الموظف 2023.xlsx](file:///c:/Users/z2k1s/Music/12522/تقرير%20تكلفة%20الموظف%202023.xlsx), سيقوم السيرفر بقراءة أوراق العمل (Sheets) ودمجها كالتالي بناءً على **"الرقم الوظيفي"** المشترك:

1. **ورقة `اسماء الموظفين` (ورقة الموظفين الأساسية):**
   * تُسخدم لقراءة البيانات الديموغرافية والوظيفية وهيكلية الشركة:
     * `الرقم الوظيفي` -> الرقم الوظيفي للموظف.
     * `اسم الموظف` -> اسم الموظف.
     * `الجنسية` -> الجنسية (سعودي أو مقيم).
     * `تاريخ بداية العمل` -> تاريخ المباشرة.
     * `المشروع` -> اسم المشروع.
     * `اسم الشركة` -> اسم الكيان الكبير (يتم إنشاؤه تلقائياً إذا لم يكن موجوداً).
     * `الرقم 700` -> الرقم الموحد 700 للكيان الكبير.
     * `الكيان` -> اسم الفرع التابع للكيان الكبير (يتم إنشاؤه وربطه تلقائياً بالكيان الكبير).

2. **ورقة `بـيـان مرتبات` (ورقة الرواتب والبدلات):**
   * يتم ربطها بالموظف عبر `الرقم الوظيفي` لقراءة البنود المالية:
     * `الراتب الأساسي` -> الراتب الأساسي.
     * `بدل السكن` -> بدل السكن.
     * `بدل الانتقال` -> بدل النقل.
     * `بدل معيشة` -> بدل المعيشة.
     * `بدلات أخرى` -> البدلات الأخرى.

3. **ورقة `تكاليف المقيمين` و `تكاليف السعودة` (ورقة الرسوم الإضافية):**
   * يتم ربطها بالموظف عبر `الرقم الوظيفي` لقراءة التكاليف الطبية والحكومية الإضافية:
     * `التأمين الطبي` -> التأمين الطبي الشهري.
     * `الشهادة الصحية` -> الشهادة الصحية الشهرية.
     * `تأشيرة الخروج والعودة` -> تأشيرة الخروج والعودة الشهرية.

## 3. تحسينات الجودة وحماية البيانات المعتمدة (Quality & Safety Enhancements)

1. **حساب السعودة التلقائي 100%:** إلغاء مبالغ السعودة اليدوية. يتم حساب السعودة لكل كيان كبير تلقائياً بجمع تكاليف موظفيه السعوديين القانونيين.
2. **حماية الحذف التراكمي:** منع حذف الكيانات الكبرى التي تتبعها فروع نشطة، ومنع حذف الفروع التي تحتوي على موظفين أو مشاريع نشطة.
3. **الفلترة الديناميكية لخيارات المشاريع:** في نموذج الموظف، بمجرد اختيار الفرع المالي، يتم فلترة قائمة المشاريع لتظهر فقط المشاريع المرتبطة بالفرع المختار.
4. **تقسيم لوحة التحكم (Compliance vs Financial View):**
   * **جدول الالتزام القانوني:** يعرض الكيانات الكبرى ونسب توطينها وعبء السعودة الناتج عنها.
   * **جدول التشغيل المالي:** يعرض الفروع وتكاليف الرواتب والمخصصات الفعلية المحملة عليها.

## 4. هيكل قاعدة البيانات المحدث (Database Schema)

* **جدول الكيانات الكبرى (`entities`):**
  * `id` INTEGER PRIMARY KEY, `name` TEXT, `unified_number` TEXT
* **جدول الفروع الجديد (`branches`):**
  * `id` INTEGER PRIMARY KEY, `name` TEXT, `cr_number` TEXT, `entity_id` INTEGER
* **جدول المشاريع المحدث (`projects`):**
  * `id` INTEGER PRIMARY KEY, `name` TEXT, `entity_id` INTEGER, `branch_id` INTEGER
* **جدول الموظفين (`employees`):**
  * `branch_id` INTEGER (الفرع القانوني)
  * `cost_branch_id` INTEGER (الفرع المالي)
  * `project_id` INTEGER (المشروع)
  * `saudi_type` TEXT (نوع التوطين: `'working'` أو `'support'`)
