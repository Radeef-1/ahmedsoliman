| البند المالي | المعادلة في ملف الإكسيل الأصلي | المعادلة في نظام الويب الحالي | التطابق |
| :--- | :--- | :--- | :---: |
| **التأمينات الاجتماعية (السعوديين)** | `=(Basic + Housing) * 22%` | `(basic_salary + housing_allowance) * gosi_saudi_rate` | **متطابق تماماً** |
| **التأمينات الاجتماعية (المقيمين)** | `=(Basic + Housing) * 2%` (أخطار مهنية) | `(basic_salary + housing_allowance) * gosi_resident_rate` | **متطابق تماماً** |
| **مخصص الإجازة شهرياً** | `=(Basic / 30) * 21 / 12` | `(basic_salary / 30) * vacation_days_per_year / 12` | **متطابق تماماً** |
| **مخصص نهاية الخدمة شهرياً** | `=(Basic / 2) / 12` (نصف راتب سنوياً) | `(basic_salary / 2) / 12` | **متطابق تماماً** |
| **مخصص تذاكر السفر شهرياً** | `=900 / 12` | `ticket_annual_cost / 12` | **متطابق تماماً** |
| **مخصص رسوم الجوازات والإقامة** | `=650 / 12` | `passport_annual_fee / 12` | **متطابق تماماً** |
| **مخصص رخصة العمل (مكتب العمل)** | `=9700 / 12` | `work_permit_annual_fee / 12` | **متطابق تماماً** |
| **تأشيرة خروج وعودة / تأمين طبي** | قيم شهرية مدخلة باليد للعمود | قيم شهرية مدخلة باليد أو مستوردة من الإكسيل | **متطابق تماماً** |

> **ميزة إضافية في الويب:** في الإكسيل، كانت أرقام الرسوم (9700، 650، 900، 22%) ثابته وصعبة التعديل. في تطبيق الويب، أصبحت هذه الأرقام **متغيرات ديناميكية مرنة** يمكن تعديلها بضغطة زر من شاشة "إعدادات التكاليف" ليعاد حساب كل شيء فوراً.

## 2. منطق عزل الكيانات وحساب السعودة (Saudization Burden logic)

في شيت الإكسيل، يتم توزيع "مبلغ السعودة" الخاص بالكيانات (الشركات الفرعية) كعبء مالي مضاف بالتساوي على الموظفين المقيمين. تم تطبيق هذا المنطق حرفياً كالتالي:

1. **احتساب إجمالي مبلغ السعودة للمجموعة (Total Saudization Cost):**
   * يقوم النظام بجمع مبالغ السعودة للشركات الفرعية (مثال: ريال البركة `54,942.25` + الرزيحي للتجارة `30,286.52` ... إلخ).
   * **تلقائية النظام:** إذا تركت مبلغ السعودة لأي كيان فرعي بقيمة `0`، يقوم النظام تلقائياً بجمع تكاليف الموظفين السعوديين الفعليين المسجلين في هذا الكيان كبديل ذكي.
2. **احتساب متوسط العبء للمقيم الواحد:**
   * يقسم النظام إجمالي المبالغ على **إجمالي عدد المقيمين النشطين بالشركة**.
3. **توزيع العبء بالتساوي:**
   * تضاف القيمة الناتجة (مثال: `+900.69` ريال) شهرياً إلى التكلفة الإجمالية لكل مقيم، وهي تطابق تماماً خلية التثبيت `$K$329` في شيت الإكسيل الأصلي.

## 3. هيكلة التقارير والمشاريع (Pivot Report Audit)

شاشة **"تقرير المشاريع"** تحاكي ورقة العمل التجميعية **"تقارير"** في الإكسيل بشكل كامل:
* **التجميع الذكي:** يتم تصنيف الموظفين حسب المشاريع/الفروع، وتجميع رواتبهم ومخصصاتهم وعبء السعودة الخاص بكل فرع على حدة.
* **المطابقة الحسابية:** مجموع التكاليف الكلية الشهرية والسنوية لجميع المشاريع في التطبيق يساوي **بالسنت الواحد** مجموع التكلفة الكلية للشركة، مما يضمن خلو الحسابات من أي فروقات تجميعية.
* **التصدير:** يتيح النظام تصدير هذه التقارير المجمعة لملفات CSV لسهولة مشاركتها أو معالجتها مالياً.
